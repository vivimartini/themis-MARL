# themis-MARL

Empirical-game stress-testing of the Themis international carbon-pricing mechanism.
This repository contains the RQ2 (strategic exploitability) experiments and figure
generation for the dissertation *"Grounding and Stress-Testing the Themis Carbon
Pricing Mechanism with Empirical-Game Methods"* (MPhil MLMI, Cambridge, 2026).

The mechanism: nine actors report willingness curves, a common carbon price is
selected by an emission-weighted quantile rule, coverage is chosen endogenously by
maximising coverage × price, and revenue is redistributed via balanced transfer
rates. At fixed coverage the price rule is strategy-proof; the endogenous coverage
step is not covered by that guarantee. These experiments measure the gap.

## Repository contents

### Data

| File | Contents |
|---|---|
| `actors_baseline.csv` | The grounded nine-actor preference layer (Chapter 3 of the dissertation). One row per actor: per-capita emissions `e`, population `pop_m`, GDP per capita, and the three preference coefficients `alpha_base` (anchored to OECD effective carbon rates), `alpha_cov` (calibrated coverage sensitivity — the declared unknown), and `alpha_trf` (transfer sensitivity, 20000/GDPpc capped at 20). Also carries the per-actor uncertainty ranges and source notes used by the scenario prior. |
| `mc_p.npz` | 4,000 Monte Carlo draws of the operating price `p` and coverage `c` under the calibration prior (RQ1, Figure 3.1). Generated by `mc_scenario_prior.py` (lives in the main dissertation repo, not here). Median 26.30, mean 26.76, 5–95% band 15.35–40.33. |
| `exA_final.npz` | Output of experiment A in `rq2_exinterim_guardrails.py`: `REG` (120 worlds × 9 actors regret matrix) and `MODE` (attack-mode labels). Consumed by `figures_rq2.py` (Figures 4 and 9). Regenerated by running the ex-interim script. |

### Core solvers and shared machinery

| File | Contents |
|---|---|
| `rq2_endogenous_coverage.py` | **The full mechanism solver and the headline experiment (RQ2 step 2).** Loads `actors_baseline.csv`, enumerates all valid coalitions (of the 512 subsets, those with ≥2 members and at least one contributor and one beneficiary), solves the self-consistent operating point on a fine 101-point transfer-rate grid (`solve_full`), and defines the two utilities (`peaked` and `transfer`). Run as a script, it computes each actor's best-response regret under endogenous coverage. Headline output: truthful point €26.70 at 86.9% coverage; peaked NashConv ≈ 19.6. |
| `rq2_fast.py` | Coarse-grid variant of the same solver (25-point transfer grid, ~4× faster), used for the distributional experiments only. Also contains the hybrid best-response oracle (`oracle`: portfolio warm starts + uniform random sampling + always-run CMA-ES refinement) and the attack-mode classifier (`attack_mode`: none / entry / exit-shrink / extortion / nudge / collapse). Import-only; no `__main__`. |
| `rq2_oracle.py` | Shared plateau-robust CMA-ES search (`cma_minimize`): explicit ask/tell loop, three restarts with widening step size, global best-ever tracking, always launched from the truthful point. Warm-start portfolios (`WARM_STARTS`, `EXIT_SHRINK_STARTS`) are evaluated as best-ever candidates only and never re-centre the search. All experiment scripts route their CMA searches through this module so seeding and budget-splitting behave identically. |

### Experiments

| File | Contents | Dissertation section |
|---|---|---|
| `rq2_fixed_coverage_control.py` | **The theorem-backed positive control (RQ2 step 1).** Freezes coverage at the exact truthful member mass and the transfer pair at the truthful solution, keeps the quantile price selection live, and runs the best-response oracle for all nine actors under both utilities. Must return NashConv = 0.0000 on the peaked domain (and does); the transfer-domain result (≈146.6) illustrates what the theorem's preference-domain condition excludes. | §5.0.1 |
| `rq2_obstruction_voteselling.py` | **Obstruction and pairwise vote-selling (RQ2 steps 3–4).** Obstruction: each actor misreports to minimise the mechanism objective c·p; reports damage and self-cost (China: 68.9% damage at no private cost). Vote-selling: for all 72 ordered (buyer, seller) pairs, maximises joint transfer surplus over the seller's report; classifies pairs as "aligned" (seller gains from its own lie) vs true purchases requiring a bribe. Headline: frontier buys China's report for a 949.6/capita surplus via the uncapped beneficiary rate. | §5.0.4 |
| `rq2_exinterim_guardrails.py` | **Three experiments in one driver.** (A) Ex-interim regret: 120 preference worlds sampled from the scenario prior, hybrid oracle per actor per world, attack-mode labels; writes `exA_final.npz`. (B) Guardrail ablation: reruns exploitability, obstruction and the two collusive deals under five designs (baseline, T⁻ ≤ 1, coverage floor ≥ 0.5, pool cap ≤ €150bn, all three). (C) Optimizer validation: hybrid oracle vs pure random search at equal budget. | §5.0.3, §5.0.5, Fig 4.2 |
| `rq2_psro_lite.py` | **Restricted empirical game with alpha-Rank (RQ2 step 8).** Seeds policy sets with each actor's truthful curve plus its discovered attack, evaluates all joint profiles exactly through the solver, computes the multi-population alpha-Rank stationary distribution across selection intensities, and runs one PSRO expansion round. Run under the baseline and the pool-cap guardrail. Headline: truthful reporting carries zero stationary mass in both designs. | §5.0.6 |

### Figures, validation, tooling

| File | Contents |
|---|---|
| `figures_rq2.py` | Generates all 12 dissertation figures into `figures/` (PDF). Reads `mc_p.npz` and `exA_final.npz`; point-calibration values (regret geography, guardrail bars, regime map) are hard-coded from the seeded experiment outputs, with provenance noted in each figure's comment. Paths are configurable via `RQ2_DATA` and `RQ2_FIGURES` environment variables. |
| `validate_rq2.py` | Fast regression suite (~20 s). Checks the truthful operating point, fixed-vs-endogenous and coarse-vs-fine solver agreement, the zero-regret control, the headline NashConv, oracle reproducibility, and China's obstruction damage. Exits non-zero on any failure. |
| `run_rq2.sh` | One-command reproduction: creates the venv if needed, then runs validation → endogenous coverage → ex-interim/guardrails → figures, in order. |
| `requirements.txt` | Pinned dependencies. **The `cma` pin matters**: discovered regrets are finite-budget search results and can shift at the second decimal across CMA-ES library versions. |

## How to reproduce the results

Requires Python 3.13 (any ≥3.10 should work; 3.13 is what the pinned versions were validated on).

### Quick start

```bash
git clone <this-repo>
cd themis-MARL
./run_rq2.sh
```

### Manual steps

```bash
# 1. Environment
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt

# 2. Sanity checks (~20 s) — all checks should PASS
.venv/bin/python validate_rq2.py

# 3. Experiments, in gating order
.venv/bin/python rq2_fixed_coverage_control.py    # ~30 s; NashConv must be 0.0000 (peaked)
.venv/bin/python rq2_endogenous_coverage.py       # ~30 s; headline NashConv ≈ 19.6
.venv/bin/python rq2_obstruction_voteselling.py   # ~1 min; 68.9% / 949.6 headlines
.venv/bin/python rq2_exinterim_guardrails.py      # ~2 min; writes exA_final.npz
.venv/bin/python rq2_psro_lite.py                 # ~30 s; truthful alpha-Rank mass 0

# 4. Figures (needs exA_final.npz from step 3)
.venv/bin/python figures_rq2.py                   # writes 12 PDFs to ./figures/
```

Note for zsh users: don't paste the trailing `#` comments into the shell — zsh
misparses `~` inside them.

### Reproducibility notes

- **Seeds.** Every stochastic component is seeded (CMA-ES per actor as 42+i, the
  ex-interim world generator at 7, random-search baselines at 3). The CMA-ES library
  treats seed 0 as unseeded, so all seeds are non-zero. Repeat runs in the same
  environment are bit-identical.
- **Oracle-dependence.** Reported regrets are finite-budget lower bounds. Exact
  values can shift at the second decimal across `cma` versions (e.g. the headline
  NashConv is 19.52–19.63 depending on library version); the actor-level geography
  (who attacks, by which mode) is stable. Use the pinned `requirements.txt` to match
  the validated values.
- **Grids.** Point-calibration headlines use the fine 101-point transfer grid;
  distributional work (ex-interim, guardrails, empirical game) uses the 25-point
  coarse grid. The two agree at the baseline (20.3 coarse vs 19.6 fine).
